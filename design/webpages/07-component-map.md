# Component Map

Use these shared components across all `Parallax Story Composer` pages to keep the experience consistent.

## Navigation

- Global navbar
- Page active state
- Auth state actions
- User avatar/menu
- `Home` nav route to `/`
- `Projects` nav route to `/projects`
- `New Project` nav route to `/projects/new`
- `Settings` nav route to `/settings`

## Buttons

- Primary neon green CTA button
- Secondary dark surface button
- Ghost button
- Danger/confirm button for delete flows
- Gated action button with subtle lock icon

## Inputs

- Text input
- Textarea
- Search input
- Select/dropdown
- Toggle
- Slider
- Read-only field / locked display input

## Cards and Panels

- Hero visual panel
- Project card
- Scene card
- Settings panel
- Auth card
- Preview stage frame
- Account section panel
- Preferences section panel

## Status UI

- Scene status chips: `Uploaded`, `Queued`, `Processing`, `Ready`, `Failed`
- Project status chips: `Draft`, `Processing`, `Renderable`
- Inline progress indicators
- Processing overlays

## Media and Workflow UI

- Upload dropzone
- Portrait preview frame
- Scene storyboard list
- Timeline hint / progress indicator
- Retry and regenerate actions
- Settings save bar or sticky actions

## Consistency Rules

- Navbar stays visually consistent across every page
- Neon green is reserved for primary emphasis, active state, ready state, and focused controls
- Dark grey surfaces should define layout hierarchy
- Portrait-first framing should appear anywhere preview content is shown
