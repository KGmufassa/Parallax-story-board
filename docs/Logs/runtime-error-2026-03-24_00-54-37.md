# Runtime Error Log

Timestamp: 2026-03-24 00:54:37

## Summary

Resolved a client-side Next.js Webpack runtime failure triggered after the grouped parallax editor changes were introduced.

## Error Observed

- Error type: `Runtime TypeError`
- Error message: `__webpack_modules__[moduleId] is not a function`
- Runtime: `Next.js 15.5.12 (Webpack)`
- Affected area: grouped parallax editor and preview client bundle

## Root Cause

- Client-side files were importing types from the barrel file `@/modules/scenes`
- That barrel re-exports runtime server modules such as `service.ts` and `validator.ts`
- Even though the imports were written as type imports, the client bundle path still resolved through the server-oriented barrel, which produced an invalid Webpack module wiring scenario at runtime

## Fix Applied

- Updated `src/features/projects/components/project-editor-page.tsx`
  - changed grouped parallax type imports from `@/modules/scenes` to `@/modules/scenes/types`
- Updated `src/features/preview/lib/playback-client.ts`
  - changed grouped parallax type imports from `@/modules/scenes` to `@/modules/scenes/types`
- This keeps the client bundle on a type-only, client-safe module path and avoids pulling the scenes barrel into the browser graph

## Validation

- `npm run build` passed
- `npm run lint` passed
- Confirmed there are no remaining `@/modules/scenes` imports inside `src/features/**`

## Files Involved

- `src/features/projects/components/project-editor-page.tsx`
- `src/features/preview/lib/playback-client.ts`

## Prevention

- Do not import from feature/module barrels inside client components when those barrels also export server runtime code
- Prefer direct type-only imports from leaf type files such as `@/modules/scenes/types`
