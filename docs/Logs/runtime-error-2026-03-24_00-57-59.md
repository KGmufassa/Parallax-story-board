# Runtime Error Log

Timestamp: 2026-03-24 00:57:59

## Summary

Investigated the persistent Next.js runtime error `Cannot find module './611.js'` and traced it to a stale mixed-mode `.next` output directory rather than a new application-code import bug.

## Error Observed

- Error type: `Runtime Error`
- Error message: `Cannot find module './611.js'`
- Require stack starts from `/.next/server/webpack-runtime.js`
- Affected route example: `/.next/server/app/projects/page.js`
- Next.js version: `15.5.12 (Webpack)`

## Root Cause

- The `.next` directory currently contains mixed artifacts from different compilation modes:
  - `/.next/server/app/projects/page.js` is a dev-style file with `eval-source-map`
  - `/.next/server/pages/_document.js` is a minified production-style file that requires chunk `611`
- That means the server runtime is reading a stale `pages/_document.js` produced by an earlier build while other route files were regenerated in dev mode
- When Webpack tries to load sibling chunk `./611.js`, the chunk graph no longer matches the stale entry that requested it, so the runtime fails

## Why The Previous Fix Was Not Enough

- The earlier client-import fix removed one bundling issue, but it did not clear the already-corrupted `.next` state
- Once `.next` contains incompatible dev/prod outputs, the app can keep failing until the build output is reset

## Fix Applied

- Updated `package.json` to clean `.next` before both development and production builds:
  - added `clean`
  - added `predev`
  - added `prebuild`
- This prevents stale chunk manifests and mixed-mode artifacts from surviving across `next dev` and `next build`

## Permanent Fix Plan

1. Stop any running Next.js dev or start processes
2. Remove the current `.next` directory once to reset the corrupted output state
3. Restart with `npm run dev` or rebuild with `npm run build`
4. Keep using the new `predev` and `prebuild` hooks so mode switches always start from a clean output directory
5. Avoid reusing `.next` across different execution modes or interrupted builds

## Validation

- Confirmed the current `.next` state shows mixed dev/prod artifacts, which matches the missing-chunk symptom
- `npm run build` passed after the script changes

## Files Involved

- `package.json`
- `.next/server/app/projects/page.js`
- `.next/server/pages/_document.js`
- `.next/server/webpack-runtime.js`

## Notes

- This class of error is usually build-output corruption, not a source-code logic error
- If it reappears after a crash or interrupted compile, a clean rebuild should recover immediately
